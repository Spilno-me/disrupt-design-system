<h2 style={{ marginTop: SPACING.px.sectionHeadingTop, marginBottom: SPACING.px.sectionHeadingBottom }}>Quick Reference</h2>

<div style={{
  display: 'grid',
  gridTemplateColumns: 'repeat(2, 1fr)',
  gap: '16px',
  marginBottom: SPACING.px.section,
}}>
  <GuidelineCard type="do" title="Make minimal changes">
    Fix only the bug. Don't refactor. Don't improve. Don't clean up. One bug = one fix.
  </GuidelineCard>

  <GuidelineCard type="dont" title="Scope creep">
    "While I'm here, I'll also..." NO. Open a separate PR or request if you see other issues.
  </GuidelineCard>

  <GuidelineCard type="do" title="Add a regression test">
    Every bug fix should include a test that would have caught the bug.
  </GuidelineCard>

  <GuidelineCard type="dont" title="Skip the test">
    "It's a simple fix" - that's exactly when bugs come back. Test it.
  </GuidelineCard>

  <GuidelineCard type="do" title="Link to reproduction">
    Point to a Storybook story that shows the bug (before) and the fix (after).
  </GuidelineCard>

  <GuidelineCard type="dont" title="Submit without context">
    PRs without reproduction steps take longer to review and may be rejected.
  </GuidelineCard>

  <GuidelineCard type="do" title="Ask if unsure">
    Not sure if your change is allowed? Ask the architect before starting.
  </GuidelineCard>

  <GuidelineCard type="dont" title="Assume and proceed">
    Don't assume design changes are ok. They're not. Always ask first.
  </GuidelineCard>
</div>
