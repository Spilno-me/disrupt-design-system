## The Paradigm Shift

| Old Paradigm | New Paradigm |
|--------------|--------------|
| Agent selects components | Agent expresses **intentions** |
| Composes layout | Constraints **shape** resolution |
| Renders output | Form **emerges** |

**The agent never says "RadioGroup" or "Select".** It expresses what the user needs to DO.

## Core Abstractions

### 1. Intention

What the user wants to **DO** â€” not what they should see.

```typescript
{
  action: "choose-one",           // The verb
  subject: {
    type: "severity",             // What kind of thing
    label: "Issue Severity"       // Human description
  },
  purpose: "request"              // Why
}
```

### 2. Constraints

The **boundaries** that shape resolution:

- **Device**: Mobile, desktop, touch, voice
- **Accessibility**: Screen reader, high contrast, reduced motion
- **Design System**: DDS tokens, density, theme
- **Context**: Urgency, available space, journey phase

### 3. Resolution

The process where intention + constraints â†’ **manifest form**.

Not component selection. **Form emergence.**

## Usage

```typescript
import {
  resolve,
  createDefaultConstraints,
  createSelectionIntention
} from '@/lib/agentic-ui';

// Create an intention
const intention = createSelectionIntention(
  [
    { value: 'critical', label: 'Critical' },
    { value: 'high', label: 'High' },
    { value: 'medium', label: 'Medium' },
  ],
  'Issue Severity',
  { required: true, type: 'severity' }
);

// Get constraints (auto-detects device, accessibility, etc.)
const constraints = createDefaultConstraints();

// Resolve!
const resolution = resolve(intention, constraints);

// Use the resolution
console.log(resolution.manifestation.pattern);  // "selection"
console.log(resolution.manifestation.traits);   // { interactive: true, ... }
console.log(resolution.reasoning.explanation);  // Human-readable why
```
