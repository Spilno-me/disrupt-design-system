<h2 style={{ marginTop: SPACING.px.sectionHeadingTop, marginBottom: SPACING.px.sectionHeadingBottom }}>Core Principles</h2>

<div style={{
  display: 'grid',
  gridTemplateColumns: 'repeat(2, 1fr)',
  gap: SPACING.px.cardGap,
  marginBottom: SPACING.px.section,
}}>
  <GuidelineCard type="do" title="Composition flows DOWN">
    Higher levels use lower levels. Pages use templates, templates use organisms, organisms use molecules, molecules use atoms.
  </GuidelineCard>

  <GuidelineCard type="dont" title="Upward dependencies">
    Never import from higher levels. Core should never import from Shared. Shared should never import from Products.
  </GuidelineCard>

  <GuidelineCard type="do" title="Respect domain boundaries">
    Product-specific components stay in Products/. If something needs to be shared, promote it to Shared/ with proper abstraction.
  </GuidelineCard>

  <GuidelineCard type="dont" title="Mix domains">
    Don't put product-specific components in Core. Don't put product dialogs in Shared unless they're truly generic.
  </GuidelineCard>

  <GuidelineCard type="do" title="Label every story">
    Include **Type:** label in every story description. Makes it easy to understand component complexity at a glance.
  </GuidelineCard>

  <GuidelineCard type="dont" title="Skip documentation">
    Don't create stories without proper domain navigation title and atomic type label. Future developers need this context.
  </GuidelineCard>
</div>
