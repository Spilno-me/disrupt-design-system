  </GuidelineCard>

  <GuidelineCard type="dont" title="Change Learned Behaviors">

  gap: SPACING.px.comfortable,
  marginBottom: SPACING.px.section,
}}>
  <GuidelineCard type="do" title="Step Back When Stuck">
    Try a different approach instead of brute-force hammering. Exploration beats forcing.
  </GuidelineCard>

  <GuidelineCard type="dont" title="Brute-Force Hammer">
    Forcing the same approach creates resistance. Step back and investigate why it's not working.
  </GuidelineCard>

  <GuidelineCard type="do" title="Break Tasks Down">
    Start with the smallest tractable piece. Make progress on that first.
  </GuidelineCard>

  <GuidelineCard type="dont" title="Plan Everything at Once">
    Huge scope creates friction. If context is growing unexpectedly, scope is too large.
  </GuidelineCard>

  <GuidelineCard type="do" title="First Draft Can Be Rough">
    Get it working first, refactor after. Functional before polished.
  </GuidelineCard>

  <GuidelineCard type="dont" title="Over-Optimize Prematurely">
    Polishing before functional wastes cycles. Make it work, then make it clean.
  </GuidelineCard>

  <GuidelineCard type="do" title="Investigate Friction">
    When pattern doesn't fit, ask WHY. Friction contains information.
  </GuidelineCard>

  <GuidelineCard type="dont" title="Ignore Warning Signs">
    Forcing through friction produces brittle code. The friction is telling you something.
  </GuidelineCard>
</div>
