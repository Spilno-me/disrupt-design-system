# Color System

The Disrupt Design System uses a **Two-Tiered Color Architecture** for simplicity and maintainability.

## The Two Tiers

  <strong>Primitives → Alias</strong>
  <p style={{ margin: '8px 0 0 0', fontSize: '14px' }}>
    Raw values → Semantic meaning (use ALIAS in components)
  </p>
</div>

### TypeScript ALIAS Tokens (For Dynamic Styles)

Use ALIAS tokens when you need **dynamic/computed values** based on state or props.

```typescript
import { ALIAS, ABYSS, DEEP_CURRENT } from '@adrozdenko/design-system'

// Dynamic styling based on state (use ALIAS)
const bgColor = isError ? ALIAS.status.error : ALIAS.background.surface
const textColor = isActive ? ALIAS.text.accent : ALIAS.text.primary

<div style={{ backgroundColor: bgColor, color: textColor }}>
  Dynamic content
</div>

// Tier 1: Raw primitives (only when needed for custom scales)
const darkScale = ABYSS[600] // Darker shade
```

### CSS Variables

```css
/* Tier 2: Alias (use these in components) */
color: var(--alias-text-primary);
background: var(--alias-bg-surface);
border-color: var(--alias-border-focus);

/* Tier 1: Primitives (rarely needed directly) */
color: var(--brand-abyss-500);
```

## Guidelines

### Do's

- Use **ALIAS tokens** for all component styling
- Reference **Primitives** only when you need raw scale values
- Use semantic names that describe intent (text.primary, background.surface)

### Don'ts

- Don't hardcode hex values - always use tokens
- Don't use Primitives directly when an ALIAS token exists
- Don't create new colors outside the system
