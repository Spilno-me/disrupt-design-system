# Corrective Actions Module - UI Design Specification

> Design system components and page layouts for the Corrective Actions module

---

## Design Principles

1. **Task-Oriented**: Focus on workflow actions and pending tasks
2. **Status Visibility**: Clear visual indicators for status, priority, and due dates
3. **Progressive Disclosure**: Show summary first, details on demand
4. **Mobile-Responsive**: Support field workers on mobile devices

## 6. Responsive Breakpoints

### Mobile (< 640px)
- Stats: 2x2 grid
- Table: Card view instead of table rows
- Forms: Single column, full width
- Filters: Collapsed into dropdown sheet

### Tablet (640px - 1024px)
- Stats: 4 across
- Table: Reduced columns (hide Description, Location)
- Forms: 2-column layout
- Filters: Visible, inline

### Desktop (> 1024px)
- Full layout as designed
- Side panel for quick view
- All columns visible

---

## 7. Accessibility Requirements

1. **Keyboard Navigation**: All interactive elements focusable
2. **Screen Reader**: ARIA labels for status badges, priority indicators
3. **Color Contrast**: WCAG AA minimum (4.5:1 for text)
4. **Focus Indicators**: Visible focus rings on all controls
5. **Error Messages**: Associated with form fields via aria-describedby
6. **Status Updates**: aria-live regions for dynamic content
